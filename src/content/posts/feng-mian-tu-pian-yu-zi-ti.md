一个写作主题如果把所有精力都放在视觉上，最后常常会输给阅读本身；但如果完全不处理视觉层，内容又容易缺乏识别度。这个主题试图做的是第三种选择：让视觉服务内容，而不是盖过内容。

## 封面图在这里承担什么角色

封面图不是为了让首页像海报墙，而是为了在文章列表里快速建立主题差异。

你可以观察当前 demo 的几个地方：

- 首页卡片会根据是否有封面自动切换布局
- 文章页封面会以更宽的比例展示
- 同一张本地图会在列表和详情页以不同尺寸输出

这些行为背后依赖的不是手写 `<img>`，而是 Astro 的资源处理能力。

## 为什么要用 `astro:assets`

封面图字段在内容集合里通过 `image()` 校验，所以：

- 路径写错时会尽早报错
- 构建期可以生成更适合前端加载的输出
- 模板里可以直接使用 `Picture`

这让图片不再是“内容文件里一段裸路径字符串”，而是真正参与构建链路的资源。

## 字体不是越多越好

这个主题默认使用偏 editorial 的字体组合，把正文和标题拉开气质差异；同时也提供可选的 WenKai 预设，用于更偏中文写作的视觉风格。

如果你想切换字体预设，只需要改配置：

```ts
export const siteConfig = {
  typography: {
    preset: "editorial",
  },
} as const;
```

可选值目前是：

- `editorial`
- `wenkai`

而且字体资源已经做成按需加载，默认 editorial 模式不会无条件打包 WenKai。

## 深浅主题为什么也算“内容功能”

对长期阅读来说，深浅主题不是花哨功能，而是阅读条件的一部分。尤其在技术博客里，代码块、表格和长文目录如果在暗色模式下层次不清，整个页面会立刻显得疲惫。

当前主题里和暗色模式直接相关的部分包括：

- 全局颜色变量
- 代码高亮主题
- 卡片、边框和玻璃态面板
- 评论区变量映射

也就是说，它不是简单换个背景色，而是整套视觉变量一起切换。

## 有封面与无封面，为什么都要在 demo 里出现

真实博客里不可能每篇文章都有封面，也不应该逼着所有作者每篇都配图。所以这次 demo 会故意混排：

- 有封面的文章
- 无封面的文章

这样你可以直接观察文章卡片是否在两种情况下都成立，而不是只在“演示最漂亮的那几篇”里好看。

## 一个写作主题的视觉边界

下面这份清单，基本就是我对这个主题视觉边界的定义：

- [x] 有明确气质，但不抢正文
- [x] 支持封面图与无封面两种写法
- [x] 深浅主题都能稳定阅读
- [x] 中文场景可以切到 WenKai
- [x] 视觉样式尽量复用，不为单个页面做过多特殊判断
